## Step-by-Step Zap Creation Guide

### Creating Your First Demo Zap (Slack)

**Step 1: Start New Zap**
```
1. Go to https://zapier.com/app/dashboard
2. Click "Create Zap"
3. Choose "TriggersAPI" as trigger
4. Click "Event Received"
5. Click "Continue"
```

**Step 2: Connect TriggersAPI Account**
```
1. Click "Connect a new account"
2. If TriggersAPI is private app:
   - Click "Sign in with your own app"
   - Enter app URL: https://your-domain.com (or localhost:8787 for dev)
   - Provide API key/auth token if required
3. Click "Continue"
```

**Step 3: Test the Trigger**
```
1. Click "Test trigger"
2. TriggersAPI will make POST to /zapier/hook to subscribe
3. Wait for "We found a sample" message
4. Sample event should show: event_id, event_type, timestamp, payload
5. Click "Continue"
```

**Step 4: Choose Action (Slack)**
```
1. Click "Do this..." button
2. Search for "Slack"
3. Choose "Send Direct Message"
4. Click "Continue"
```

**Step 5: Connect Slack Account**
```
1. Click "Connect a new account"
2. Authorize Zapier in Slack workspace
3. Click "Continue"
```

**Step 6: Configure Slack Message**
```
1. Message text: Click "+" icon and insert fields from TriggersAPI trigger:
   - {event_id}
   - {event_type}
   - {timestamp}
   - {payload}
2. Format as: "Event #{event_id} | Type: {event_type} | Time: {timestamp}"
3. To: Choose your Slack user
4. Click "Continue"
```

**Step 7: Test Action**
```
1. Click "Test action"
2. Zapier will send test Slack message
3. Verify message appears in Slack
4. Click "Continue"
```

**Step 8: Enable Zap**
```
1. Give Zap name: "TriggersAPI ‚Üí Slack Notification"
2. Click "Publish"
3. Zap is now live!
```

**Step 9: End-to-End Test**
```
1. Send event to TriggersAPI:
   curl -X POST http://localhost:8787/events \
     -H "Authorization: Bearer your-token" \
     -H "Content-Type: application/json" \
     -d '{
       "event_type": "payment_received",
       "payload": {"amount": "$100", "customer": "John Doe"}
     }'
2. Check Slack - message should appear in 5-10 seconds
3. Verify all event details present
```

## Troubleshooting Guide

**Issue: "Test trigger" fails with "No sample found"**
- TriggersAPI /zapier/hook endpoint not responding
- Check backend is running: `npm run dev`
- Check network connectivity if using remote URL
- Verify webhook storage working (query zapier_webhooks table)

**Issue: Slack message not arriving**
- Check Zapier logs for delivery errors
- Verify Slack authorization still valid
- Check Slack workspace has Zapier app installed
- Verify Slack user ID is correct in action config

**Issue: Email not arriving**
- Check Gmail account has IMAP enabled
- Verify Gmail app authorization not revoked
- Check email spam folder
- Verify email address in action config is correct

**Issue: Notion database not updated**
- Verify Notion database exists and is shared with Zapier
- Check database schema matches field mapping
- Verify field types compatible (text for text, date for date)
- Try creating entry manually in Notion to test permissions

**Issue: Events not flowing through any Zap**
- Check webhook subscription created: query D1 `zapier_webhooks` table
- Verify webhook status = 'active' (not 'failing')
- Check TriggersAPI logs for delivery errors
- Test event delivery with curl to webhook URL directly
