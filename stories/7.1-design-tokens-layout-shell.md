## Summary

Establish the visual foundation and structural scaffold for the Mission-Control Pulse dashboard. Implement comprehensive design tokens (Zapier palette, Degular/Inter typography, 8px spacing grid, elevation system), configure Tailwind CSS with custom token extensions, and build the two-column responsive layout shell that serves as the foundation for all subsequent dashboard stories. This story ensures all components downstream inherit consistent, on-brand styling and responsive behavior that supports desktop-first Mission-Control experience with tablet/mobile fallbacks.

## Business Value

Design tokens and layout shell are the foundation for visual consistency, brand alignment, and responsive behavior across the entire dashboard. By establishing these standards early, all downstream stories can inherit consistent styling without reimplementation, reducing development time and ensuring cohesive user experience. Zapier-branded theme codifies design authority and demonstrates product excellence. Responsive layout shell ensures dashboard works across all devices while maintaining desktop-first philosophy for demo scenarios. Early accessibility infrastructure (color contrast, spacing consistency) prevents rework downstream.

## User Stories

**As a** developer implementing dashboard components

**As a** demo facilitator

**So that** stakeholders see the structural vision immediately and components slot in naturally

- Alert Error: `--color-alert-error` = `#FF4F00` (Zap Orange) with `#201515` base — Failure chips, error logs
- Neutral 100: `--color-neutral-100` = `#FFFDF9` (Almost White) — Primary text on dark surfaces
- Neutral 400: `--color-neutral-400` = `#FFF3E6` (Cream) — Secondary text, separators

**Typography System (Section 3.2):**
- Headlines (H1/H2/H3): Degular (brand primary) — Display Bold (H1 2.5rem), Medium (H2 1.75rem, H3 1.5rem)
- Body/Data/Logs: Inter (sans-serif) 16px base with tabular numerals for IDs/latency
- Mono (logs/code): JetBrains Mono fallback for code snippets
- Line Heights: 1.2 headlines, 1.45 body
- Type Scale: H1 2.5rem, H2 1.75rem, H3 1.5rem, Body 1.0rem, Caption 0.875rem, Mono 0.875rem

**Spacing & Layout (Section 3.3):**
- Base Unit: 8px (Tailwind `2`)
- Spacing Scale: `xs=4px`, `sm=8px`, `md=12px`, `lg=16px`, `xl=24px`, `2xl=32px`, `3xl=48px`
- Grid: 12-column at ≥1440px, 8-column at ≥1024px, stacked <768px
- Container Max: 1440px; hero rail `min-width: 340px`
- Elevation: `card-base` (1px border + subtle shadow), `card-active` (glow + 4px shadow), `modal` (16px blur)

**Layout Direction (Section 4.1: Mission-Control Pulse):**
- Primary Canvas (Column 1): Hero Run Command panel (top, sticky), Event Timeline below, spanning full height
- Secondary Canvas (Column 2): Stacked collapsible panels (Logs, Inbox, Metrics, History)
- Desktop (≥1440px): Two-column layout side-by-side

- Tablet (768–1199px): Columns stack vertically, controls top-anchored

## Implementation Notes

### Approach
1. **Establish tokens first:** Design tokens are immutable contracts that downstream components inherit from. Implement before layout so layout can reference tokens cleanly.
2. **Configure Tailwind:** Extend Tailwind config to consume token definitions, avoiding duplication.
3. **Build layout shell:** Create responsive layout structure without component content; use placeholder divs or stubs.
4. **Document & showcase:** Storybook and design token README enable self-service for downstream developers.
5. **Test comprehensively:** Verify tokens are accessible, layout is responsive, accessibility is solid.

### Key Decisions
- **CSS Custom Properties vs Tailwind:** Tokens defined as TypeScript constants AND CSS custom properties, allowing both programmatic access and dynamic theming later.
- **Responsive-First:** Use Tailwind responsive prefixes (md:, lg:, xl:, 2xl:) for all layout changes; avoid JavaScript-based breakpoint detection.
- **Sticky Run Command:** Column 1 Run Command panel remains sticky during scroll; Timeline scrolls independently within column.
- **Collapsible Panels:** Column 2 panels use accordion pattern (Radix UI primitive) to respect mobile space constraints.
- **Dark Mode:** Implement `prefers-color-scheme` media query for system theme detection; optional manual toggle in later story.

### Risk Mitigation
- **Font Loading Delay:** Implement system fallback fonts; use `font-display: swap` to prevent text from disappearing.
- **Layout Shift (CLS):** Reserve space for sticky Run Command and use fixed grid dimensions to prevent shifts.
- **Color Contrast on Dark Background:** Verify all colors against Night (#2B2358) and Earth (#201515) backgrounds.
- **Responsive Bugs:** Test on actual devices (not just browser emulation) for touch interactions and viewport behaviors.
